📘 مشروع 12: Build & Deploy

🎯 هتتعلم إيه في المشروع ده؟

  • إزاي تبني تطبيق React للبرودكشن
  • تجهز ملفات المشروع للنشر
  • تنشر التطبيق على Netlify أو GitHub Pages
  • تحسن أداء التطبيق وتقلل الحجم
🛠️ خطوات إعداد المشروع
  1. 1اتأكد إن مشروعك شغال كويس وجاهز للنشر.
  2. 2افتح التيرمنال في فولدر المشروع واكتب:
    3. Terminal 
    npm run build
  3. 3هتلاقي فولدر جديد اسمه dist فيه ملفات البرودكشن.
💡 ملفات dist هي اللي هتنشرها على أي سيرفر أو خدمة استضافة.
📂 هيكلية الفولدرات بعد البناء
my-react-app/
├── dist/
│   ├── assets/
│   ├── index.html
│   └── ...
├── src/
├── package.json
└── ...
✍️ خطوات النشر على Netlify
  1. ادخل على Netlify وسجل حساب.
  2. اضغط "Add new site" واختر "Deploy manually".
  3. اسحب فولدر dist وحطه في المكان المخصص على Netlify.
  4. استنى شوية وهتلاقي لينك لموقعك الجديد!
✅ كده تطبيقك بقى أونلاين!
✍️ خطوات النشر على GitHub Pages
  1. اتأكد إن عندك حساب على GitHub وعملت repository جديد.
  2. ثبت أداة gh-pages:
    3. Terminal 
    npm install --save-dev gh-pages
  3. ضيف في package.json:
    4. package.json 
    "homepage": "https://USERNAME.github.io/REPO-NAME"
  4. ضيف سكريبتات النشر:
    5. package.json 
    "scripts": {
  ...
  "predeploy": "npm run build",
  "deploy": "gh-pages -d dist"
}
  5. اعمل push للكود على GitHub.
  6. شغل الأمر ده للنشر:
    7. Terminal 
    npm run deploy
  7. استنى شوية وهتلاقي موقعك على GitHub Pages!
✅ كده تطبيقك بقى أونلاين على GitHub Pages!
🧠 المفاهيم الأساسية
  • Build: يعني تجهز ملفات التطبيق للنشر (تصغير وتحسين).
  • dist: فولدر فيه كل الملفات الجاهزة للنشر.
  • Deployment: يعني ترفع التطبيق على الإنترنت.
  • Optimization: تصغير الصور والكود عشان السرعة.
💬 شوية نصايح / أحسن ممارسات
  • جرب تبني وتنشر أكتر من مرة عشان تتعود على الخطوات.
  • راجع ملفات dist قبل النشر وتأكد إن كل حاجة شغالة.
  • استخدم أدوات زي Lighthouse عشان تقيس أداء الموقع.
  • لو فيه صور أو ملفات كبيرة، حاول تصغرها قبل النشر.
